## タスク: 伏線の追跡と管理

`summaries/` 以下の全ファイルの「## 伏線」セクションを走査し、伏線の状態を一覧表示する。

### デフォルト動作（引数なし）

すべての伏線を以下の3カテゴリに分類して表示する:

```
🔮 伏線トラッカー
═══════════════════

【未回収】 (N件)
  ch001 - エレンの冒頭の「長い夢」の内容
  ch001 - グリシャの不在と「地下室」の約束
  ch001 - 超大型巨人の正体
  ...

【回収済み】 (N件)
  ch001 → ch086 - グリシャの地下室の秘密
  ...

【部分回収】 (N件)
  ch001 → ch033(部分) - 超大型巨人の正体 → ライナーの示唆あり、確定は後
  ...
```

### 引数付き動作

- `$ARGUMENTS` = 話数（例: `ch034`）
  → その話数時点で未回収の伏線のみを表示
  → 「ch034時点であなたが抱えている謎」を再現

- `$ARGUMENTS` = `resolve ch086 "地下室の秘密"`
  → 指定した伏線を「回収済み」に更新
  → 該当するsummaryの伏線セクションを `→ ch086で回収` に書き換え

- `$ARGUMENTS` = `add ch045 "新しい伏線の説明"`
  → ch045のsummaryに新しい伏線を追記

### 注意事項
- 伏線の回収判定はユーザーと相談して決める（自動判定しない）
- 回収時は関連するWikiページ（character, event等）にも「伏線回収」の旨を追記する
- 操作後は `log.md` に記録する
